Digital Twin Model Export

The Model Export dialog box is used to export the current model into three types of Digital Twin simulation apps; Stream, Batch and Functional Mock-up Unit (FMU).

  • File Name: - For each app type, you can click the Browse... button to select the name of the zip-archive that will contain the exported model.
  • Input Indicator Group: - Specifies the name of the group of input indicators that will be used in the exported Stream- or Batch app. The names specified here need to be reflected in the cloud simulation setup in EPD Connected Products.
  • Output Indicator Group: - Specifies the name of the group of output indicators that will be used in the exported Stream app. The name specified here needs to be reflected in the cloud simulation setup in EPD Connected Products.
  • Window Size [sec]: - Specifies the length of each time window (in seconds) for the Stream app.
  • Transfer solver state between windows - Activate this toggle if each time window should be restarted using the final state of the previous window as starting point. If not activated, each time window will start from the modeling configuration.
  • Surface Only - Activate this toggle to represent all FE Parts in the model by surface elements only, in the visualization model generated by the Batch app. If not activated, the visualization model may be larger since it will also contain (invisible) internal element interfaces between the solid finite elements.
  • Stress Recovery - Activate this toggle to let the Batch app perform deformation- and von Mises stress recovery for those FE Parts that have the Perform stress recovery during dynamics simulation toggle enabled in the Advanced tab of the property editor panel.
  • All FE-Parts - Activate this toggle to let the Batch app generate a visualization for all FE Parts in the model. If not activated, only the Parts having the Perform stress recovery during dynamics simulation toggle enabled in the Advanced tab of the property editor panel will be included in the visualization. When the All FE-Parts toggle is enabled, those FE Parts which are not toggled for recovery will be represented by their rigid body motion only.
  • Input Indicators: - This table lists all External functions in the model (their Tag and Description, respectively. They will define the content of the named Input Indicator Group.
  • Output Indicators: - This table lists all Functions in the model, that have the Use as output sensor toggle enabled in the property editor panel. Their Tag and Description, respectively, are listed, in addition to the description of the Threshold settings, if that is enabled.

Pressing the Export button will perform the model export for all of the three app types which have the respective activation toggle enabled. The exported zip-file will contain all files related to the current model, in addition to some support files for executing the model in the cloud. The FMU will in addition contain shared object libraries interfacing the dynamics solver based on the Functional Mock-up Interface (fmi-standard.org), such that it can be executed on both Windows and Linux systems.
